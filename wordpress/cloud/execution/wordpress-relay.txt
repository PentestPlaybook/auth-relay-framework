1. Run wordpress-relay.sh

root@digital-ocean-hostname:~# wget https://raw.githubusercontent.com/PentestPlaybook/auth-relay-framework/refs/heads/main/wordpress/cloud/execution/scripts/wordpress-relay.sh -O wordpress-relay.sh

root@digital-ocean-hostname:~# chmod +x wordpress-relay.sh

root@digital-ocean-hostname:~# ./wordpress-relay.sh --domain your-domain.info

------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
2. Open the VNC Session

üåêVNC URL: http://docker droplet public ip:6901/?password=randomly generated password

Hit "Enter" once connected. Selenium will open firefox to the target site in your VNC Session.

------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
3. Get the REQUIRED_TOKEN value from the Environmental Variables

root@digital-ocean-hostname:~# cat.env | grep REQUIRED_TOKEN

------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
4. Visit the URL with the required token value appended to the URL

https://your-domain.info/link?src=required token value

(The REQUIRED_TOKEN value will rotate and tracker.service will restart - resetting the token value.)

Note: The target user will retain access to our web application, until they successfully enter their credentials/MFA code.

------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
5. Submit the target user's username/password and 6-digit MFA code (if applicable)

(This script works on both MFA-enabled sites and on sites that do not have MFA enabled.)

(Note: If the login fails in victim's browser but succeeds in attacker's browser, ensure miniOrange 2-factor authentication Setup has been set up / all the instructions in wordpress-setup.txt are complete.)

Monitor the wordpress-relay.sh execution output to view intercepted credentials and/or MFA code in real-time and troubleshoot any login issues.

------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
6. Monitor the Firefox Browser within the VNC Session

Once the required login information is entered, Selenium will log the attacker's firefox browser into the victim's account.

Upon successful login in attacker's browser, the victim's broswer is redirected to the target website.

The target user loses access to the web application once they successfully enter their credentials/OTP code.
