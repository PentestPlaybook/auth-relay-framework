3. Get the REQUIRED_TOKEN value from the Environmental Variables

root@digital-ocean-hostname:~# cat.env | grep REQUIRED_TOKEN

------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
4. Visit the URL while appending the required token

https://your-domain.info/link?src=required token value

The required token value will change and the service will restart. 
However, the target has session cookies that will keep them authenticated.

------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
5. Submit the username/password and 6-digit MFA code (if applicable)

(This script works on MFA-enabled sites and on sites that do not have MFA enabled.)

If login fails in victim's portal but succeeds in attacker's VNC window, ensure miniOrange 2-factor authentication Setup has been set up and wordpress-setup.txt has been completed.

Monitor the wordpress-relay.sh execution output to view intercepted credentials / MFA code in real-time and troubleshoot any issues.

------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
6. Monitor the Firefox Browser in the VNC Session

Once the required login information is entered, Selenium will log the attacker into victim's account.
Upon successful login in attacker's browser, victim's broswer is redirected to the target website.
