# Security Policy

## 1. Supported Research Scope
This project supports **authorized, good-faith security research** including:

- authentication testing in controlled environments  
- MFA resilience evaluation  
- research on TLS validation behaviors  
- detection engineering practice  
- red/blue team collaboration  
- academic or IRB-approved research

Unauthorized use is strictly prohibited.

# 3. Disclosure Process
We follow responsible disclosure:

1. Confirm receipt  
2. Reproduce issue  
3. Discuss scope and severity  
4. Fix the issue  
5. Publicly credit researcher (optional)  

---

# 4. Safe Harbor
This project aligns with:

- **Disclose.io Safe Harbor guidelines**  
- **DOJ 2022 CFAA Good-Faith Security Research Guidance**  
- **HackerOne & Bugcrowd safe-harbor principles**

Researchers acting in good faith, within authorization, and without causing harm receive full safe harbor protections.

---

# 5. Security Requirements for Users
Users must adhere to:

- SOW & ROE  
- authorization boundaries  
- data minimization principles  
- secure credential handling  
- immediate sanitization of sensitive outputs  

---

# 6. Operational Security Guidance
Recommended controls when testing:

- isolated lab networks  
- non-production systems  
- unique testing credentials  
- separate monitoring rules during tests  
- immediate data deletion post-engagement  
- logging appropriate for SOC 2 & ISO 27001 compliance  

---

# 7. Vulnerability Handling
If vulnerabilities are discovered **in target systems** during authorized tests:

- limit exploitation to proof-of-concept  
- avoid accessing personal data  
- avoid causing downtime  
- follow responsible disclosure agreements
